# Serenity × A股映射 Skill 设计文档

**日期：** 2026-06-02  
**状态：** 已批准（含社区 skill 整合更新）

---

## 背景

Serenity（@aleabitoreddit）是 X 上专注 AI 基础设施与半导体供应链的知名分析师，擅长用"卡脖子理论"发现价值洼地。用户希望每次调用时获取她的最新观点，并自动映射到 A 股对标标的，含行情与估值参考。

本 skill 整合了以下社区已有成果：
- **siisee11/serenity-skill**：6步研究流程 + 证伪清单 + 9维打分体系（方法论骨架）
- **w894781950-netizen/Serenity.skill**：7因子A股打分 + L1/L2/L3三层分级 + 游资情绪 + 风控红线（A股适配层）
- **muxuuu/serenity-skill**：7层供应链拆解 + 伪趋势识别规则（趋势甄别层）

---

## 目标

构建一个 Claude Code skill（SKILL.md），激活时：
1. 通过 WebSearch 获取 Serenity 最新帖子和观点
2. 识别真/伪超级趋势，提取核心标的及逻辑链
3. 预判她下一步关注焦点
4. 用 L1/L2/L3 分层 + 7因子打分映射 A 股对标
5. 证伪检查，过滤弱逻辑候选
6. 联动 `a-share-data` 查询行情与估值，输出结构化结论

---

## 范围

- **包含**：SKILL.md，无脚本，无定时任务，无 X API 依赖
- **不包含**：定时推送、邮件/微信通知、历史数据归档

---

## 架构

纯 SKILL.md 驱动，无外部脚本。激活时 Claude 全程在对话中完成数据拉取与分析。

```
用户调用 skill
    ↓

    ↓

```

## 激活流程（SKILL.md 内部指令）

### 第一步：拉取数据

并行执行：
- `aleabitoreddit site:x.com latest 2026`
- `serenity aleabitoreddit supply chain chokepoint new thesis 2026`
- `serenity aleabitoreddit new stock pick OR new focus 2026`

取最近 2 周内的内容，忽略无实质内容的转发。

### 第二步：真/伪超级趋势甄别

对每个 Serenity 关注的主题，判断：

| 判断 | 条件 |
|------|------|
| ✅ 真趋势 | 有物理供应链 + 有不可替代环节 + 有新品/新催化剂 |
| ❌ 伪趋势 | 纯资金轮动（如电力避险）、名字炒作、无物理约束 |

真趋势跌了是错杀（潜在买点），伪趋势涨了是陷阱（忽略）。

### 第三步：供应链7层拆解 + L1/L2/L3 分级

对每个确认的真趋势，拆解物理供应链：

**7层架构**（muxuuu框架）：
下游需求 → 系统集成 → 芯片/器件 → 设备 → 材料 → 封测 → 基础设施

**L 分级**（w894781950 框架）：
- **L1**：共识层，已充分定价（如GPU、光模块龙头）
- **L2**：发现中，正在被定价（如先进封装、高多层PCB）
- **L3**：隐蔽瓶颈，尚未被发现（核心目标，对标 Serenity 的选股逻辑）

重点关注 L3，避免追 L1。

### 第四步：预判下一焦点

基于以下维度推断：
- Serenity 近期提及频率上升最快的主题
- 她已确认的卡脖子环节的自然上下游延伸
- 已被市场炒过的 L1/L2 对应的更上游 L3 材料/设备

### 第五步：A股映射 + 7因子打分

对每个 L2/L3 候选，找国内对应公司并打分：

**7因子打分体系**（来自 w894781950/Serenity.skill v2.0）：

| 因子 | 权重 | 说明 |
|------|:----:|------|
| 不可替代性 | ×2 | 全球/国内有几家能做？客户换掉成本？ |
| 产能弹性 | ×1 | 扩产速度、现有产能是否受限 |
| 客户验证 | ×1 | 是否实质进入核心供应链（有无认证/订单证据）|
| 稀释风险 | ×1 | 定增/大股东减持/质押比例 |
| 地缘风险 | ×1 | 出口管制/制裁敞口 |
| 估值合理性 | ×1.5 | PE水位 + 30日涨幅透支度（>50%透支，<20%充裕）|
| 产业阶段 | ×1 | 催化剂时间窗口（刚启动5分/进行中3分/远期1分）|

**总分判断**：>25分优先研究，15-25分观察，<15分忽略。

### 第六步：证伪检查（siisee11 falsification checklist）

在给出看多结论前，主动搜索以下反驳证据：
- 有无替代技术且认证周期更短？
- 有无更大竞争者正在扩产？
- 客户是否在双供应商策略或内部化该组件？
- 收入敞口是否太小（< 20%收入）无法移动业绩？
- 毛利率是否与声称的定价权一致？
- 是否有频繁定增/转债/减持记录？

若 2 条以上触发，降级处理或放弃该候选。

### 第七步：行情验证 + 游资情绪

调用 `a-share-data` 获取每个候选：
- 实时股价、近1月/3月涨跌幅
- PE/PB 估值（与历史分位对比）
- 主力资金净流入
- **换手率分析**（游资情绪指标）：
  - >25%：可能出货，避免追入
  - 3-8%：正常，可关注
  - <3%：机构未介入，过早或低关注度

### 第八步：输出

```markdown
## Serenity 最新动态（YYYY-MM-DD）

### 趋势判断
| 主题 | 判断 | 理由 |
|------|------|------|
| AI CPO | ✅ 真趋势 | 物理约束明确（铜互连带宽极限），NVIDIA GB300催化 |
| 电力 | ❌ 伪趋势 | 资金避险轮动，无新产业催化剂 |

### 核心关注 → A股映射
| 海外标的 | 环节 | L级 | A股对标 | 代码 | 7因子得分 | 现价 | 30日涨跌 | PE | 换手率 | 结论 |
|---------|------|-----|--------|------|----------|------|---------|-----|------|------|

### 证伪记录
（对哪些候选进行了降级及原因）

### 预判下一焦点
（逻辑推导：L1→L2→L3 溯源路径）

### 风控检查
- [ ] 大盘今日跌幅 <1.5%
- [ ] 两市成交 >7000亿
- [ ] 候选标的换手率 <25%
- [ ] 是否周五（若是，提示减仓注意）
```

## 降级处理

| 场景 | 处理方式 |
|------|---------|
| WebSearch 无近期结果 | 使用已知历史观点兜底，注明数据日期 |
| 搜索结果无实质内容 | 告知用户，建议直接访问 x.com/@aleabitoreddit |
| a-share-data 调用失败 | 输出标的名称和代码，行情数据标注"待查" |
| 所有候选证伪检查均未通过 | 输出"当前无高置信度L3标的"，列出观察名单 |

## 成功标准

- 调用后输出完整结论，含趋势判断、L分级、7因子打分、证伪记录
- 至少识别出 1 个 L3 候选（而非只给 L1 共识股）
- 行情数据来自 a-share-data 实时查询（非静态）
- 预判焦点有 L1→L2→L3 溯源路径，非凭空列举
- 证伪步骤有实际执行记录（不能跳过）
