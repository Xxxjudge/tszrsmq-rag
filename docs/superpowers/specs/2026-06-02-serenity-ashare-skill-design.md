# Serenity × A股映射 Skill 设计文档

**日期：** 2026-06-02  
**状态：** 已批准

---

## 背景

Serenity（@aleabitoreddit）是 X 上专注 AI 基础设施与半导体供应链的知名分析师，擅长用"卡脖子理论"发现价值洼地。用户希望每次调用时获取她的最新观点，并自动映射到 A 股对标标的，含行情与估值参考。

---

## 目标

构建一个 Claude Code skill（SKILL.md），激活时：
1. 通过 WebSearch 获取 Serenity 最新帖子和观点
2. 提取核心标的及逻辑链
3. 预判她下一步关注焦点
4. 映射到 A 股对标，联动 `a-share-data` 查询行情与估值
5. 输出结构化结论

---

## 范围

- **包含**：SKILL.md，无脚本，无定时任务，无 X API 依赖
- **不包含**：定时推送、邮件/微信通知、历史数据归档

---

## 架构

纯 SKILL.md 驱动，无外部脚本。激活时 Claude 全程在对话中完成数据拉取与分析。

```
用户调用 skill
    ↓
WebSearch：拉取 @aleabitoreddit 最新动态
    ↓
提取：标的 / 逻辑 / 情绪 / 新增关注
    ↓
推断：下一焦点预判
    ↓
A股映射：卡脖子框架匹配国内对标
    ↓
（可选）联动 a-share-data：实时行情 + 估值
    ↓
输出结构化结论
```

---

## 激活流程（SKILL.md 内部指令）

### 第一步：拉取数据

执行以下搜索（并行）：
- `aleabitoreddit latest posts 2026`
- `serenity supply chain chokepoint new thesis 2026`
- `@aleabitoreddit site:x.com`

取最近 2 周内的内容，忽略转发/无实质内容的帖子。

### 第二步：提取观点

从搜索结果中识别：
- **核心持仓/关注标的**：ticker + 逻辑一句话摘要
- **新增关注**：本周新提及但之前未出现过的标的/主题
- **情绪变化**：加仓信号 / 减仓信号 / 中性观察
- **逻辑链关键词**：卡脖子环节名称（如 InP、CW laser、稀土等）

### 第三步：预判下一焦点

基于以下维度推断：
- 近期提及频率上升最快的主题
- 她的供应链溯源逻辑自然延伸的上下游
- 尚未被市场充分定价（低关注度 + 供给约束明确）

### 第四步：A股映射

对每个海外标的，用 Serenity 的 4 问快滤找国内对应：
1. 是否稀缺（国内唯一 / 寡头）？
2. 需求是否确定（AI/算力建设不可逆）？
3. 供给约束是否真实（产能、技术壁垒）？
4. 是否有催化剂（扩产、认证、政策）？

调用 `a-share-data` 获取：实时股价、近期涨跌幅、主力资金流向、PE/PB 估值。

### 第五步：输出

```markdown
## Serenity 最新动态（YYYY-MM-DD）

### 当前核心关注
| 海外标的 | 卡脖子环节 | 核心逻辑 | A股对标 | 代码 | 现价 | 近1月涨跌 | PE | 判断 |
|---------|-----------|---------|--------|------|------|----------|----|----|

### 新增关注 / 情绪变化
（本周新出现的主题或信号）

### 预判下一焦点
（逻辑推导，非猜测）

### 操作参考
（基于映射结果的简短建议，非投资建议）
```

## 与现有 Skill 的联动

| Skill | 联动方式 |
|-------|---------|
| `a-share-data` | 查询对标股票实时行情、资金流向、PE/PB |
| `tszrsmq` | 可选：用唐史视角交叉验证 A 股标的的市场逻辑 |

联动为可选，不强依赖——若 a-share-data 调用失败，仍输出映射结论，标注"行情数据待查"。

---

## 降级处理

| 场景 | 处理方式 |
|------|---------|
| WebSearch 无近期结果 | 使用已知历史观点兜底，注明数据日期 |
| 搜索结果无实质内容 | 告知用户，建议直接访问 x.com/@aleabitoreddit |
| a-share-data 调用失败 | 输出标的名称和代码，行情数据标注"待查" |

## 成功标准

- 调用后 2 分钟内输出完整结论表格
- A股对标每个标的至少给出 4 问快滤评分
- 行情数据来自 a-share-data 实时查询（非静态）
- 预判焦点有明确逻辑依据，不是凭空列举
